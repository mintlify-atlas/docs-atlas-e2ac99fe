Tabsdata brings the pub/sub pattern to tabular data. Define publishers that ingest data from files, databases, and cloud storage, write transformers that reshape it using a Polars-backed TableFrame API, and attach subscribers that push results to any destination — all wired together automatically.

## How Tabsdata Works

Tabsdata models data movement as three function types — publishers, transformers, and subscribers — registered with Python decorators and executed by the Tabsdata server.

<Steps>
  <Step title="Install Tabsdata">
    Install the package from PyPI and start the local server.
    ```bash
    pip install "tabsdata[all]"
    tdserver start
    ```
  </Step>
  <Step title="Define a Publisher">
    Decorate a Python function to publish a CSV file into a Tabsdata table.
    ```python publisher.py
    import tabsdata as td

    @td.publisher(
        source=td.LocalFileSource("./data/persons.csv"),
        tables="persons",
    )
    def pub(persons: td.TableFrame) -> td.TableFrame:
        return persons
    ```
  </Step>
  <Step title="Transform the Data">
    Apply filters and projections in a transformer that subscribes to the `persons` table and writes derived tables.
    ```python transformer.py
    import tabsdata as td

    @td.transformer(
        input_tables="persons",
        output_tables=["spanish", "french"],
    )
    def tfr(persons: td.TableFrame):
        spanish = persons.filter(td.col("nationality").eq("Spanish"))
        french  = persons.filter(td.col("nationality").eq("French"))
        return spanish, french
    ```
  </Step>
  <Step title="Register and Trigger">
    Register your functions with the CLI, then trigger execution.
    ```bash
    td fn register --coll myproject --path publisher.py --name pub
    td fn register --coll myproject --path transformer.py --name tfr
    td fn trigger --coll myproject --name pub
    ```
  </Step>
</Steps>
