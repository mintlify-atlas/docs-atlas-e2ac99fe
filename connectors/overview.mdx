Connectors bridge the gap between Tabsdata and the external world. A **source connector** is used inside a `@publisher` decorator to pull data from an external system into a Tabsdata collection, while a **destination connector** is used inside a `@subscriber` decorator to push processed data back out to an external system. Every built-in connector is a subclass of either `SourcePlugin` or `DestinationPlugin`, which means you can also write your own custom connectors that follow the same interface.

## Custom connectors

If none of the built-in connectors meet your needs, you can write your own by subclassing `SourcePlugin` or `DestinationPlugin` from `tabsdata`.

```python
from tabsdata import SourcePlugin, DestinationPlugin
```

- **`SourcePlugin`** — implement `chunk(working_dir)` to download data and save it as Parquet files, then return their paths. The framework maps each returned path to a positional argument of your publisher function.
- **`DestinationPlugin`** — implement `stream(working_dir, *results)` or the lower-level `chunk` / `write` pair to persist the TableFrame data your subscriber produces.

See [Custom Plugins](/connectors/custom-plugins) for complete examples.

<Note>
Credentials for any connector can be stored securely using `EnvironmentSecret`, `HashiCorpSecret`, or other secret types. See [Secrets](/guides/secrets) for details.
</Note>
