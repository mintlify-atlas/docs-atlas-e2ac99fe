Tabsdata uses role-based access control (RBAC). Users are accounts that can log in to the server; roles are named sets of permissions that can be assigned to one or more users. The `td user` command group manages user accounts, while the `td role` command group manages roles, their permissions, and their membership.

## td user

### `td user create`

Create a new user account.

```bash
td user create --name <name> --full-name <full-name> --email <email> \
  --password <password> [--disabled]
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Login username. |
| `--full-name` | `-f` | No* | Full display name. Defaults to `--name` if not provided. |
| `--email` | `-e` | No* | Email address. |
| `--password` | `-p` | Yes* | Password. Prompted (hidden, with confirmation) if omitted. |
| `--disabled` | | No | Flag. Create the account in a disabled state. |

\* Prompted interactively if not supplied and `--no-prompt` is not set.

**Example**

```bash
# Interactive password prompt
td user create --name alice --full-name "Alice Smith" --email alice@example.com

# Fully non-interactive
td user create \
  --name alice \
  --full-name "Alice Smith" \
  --email alice@example.com \
  --password s3cr3t
```

### `td user delete`

Delete a user account. Requires explicit confirmation.

```bash
td user delete --name <name> --confirm delete
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Username to delete. |
| `--confirm` | | Yes* | Must be the literal string `delete`. Prompted if omitted. |

**Example**

```bash
td user delete --name alice --confirm delete
```

### `td user update`

Update one or more attributes of an existing user.

```bash
td user update --name <name> [--full-name <name>] [--email <email>] \
  [--password <pw>] [--enabled <true|false>]
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Username to update. |
| `--full-name` | `-f` | No | New full display name. |
| `--email` | `-e` | No | New email address. |
| `--password` | `-p` | No | New password. |
| `--enabled` | | No | `true` to enable the account, `false` to disable it. |

**Example**

```bash
# Update email only
td user update --name alice --email alice.smith@example.com

# Disable an account
td user update --name alice --enabled false
```

## td role

### `td role create`

Create a new role.

```bash
td role create --name <name> [--description <desc>]
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Name of the role to create. Uses pinned role if omitted. |
| `--description` | | No | Human-readable description. |

**Example**

```bash
td role create --name data-engineer --description "Data engineering team role"
```

---

### `td role delete`

Delete a role. Requires explicit confirmation.

```bash
td role delete --name <name> --confirm delete
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Name of the role to delete. |
| `--confirm` | | Yes* | Must be the literal string `delete`. |

**Example**

```bash
td role delete --name old-role --confirm delete
```

---

### `td role info`

Display details of a role.

```bash
td role info --name <name>
```

Prints a table with columns: **Name**, **ID**, **Description**, **Created on**, **Created by**.

**Example**

```bash
td role info --name data-engineer
```

---

### `td role list`

List all roles on the server.

```bash
td role list
```

**Example**

```bash
td role list
```

---

### `td role update`

Update the name and/or description of a role.

```bash
td role update --name <name> [--new-name <new-name>] [--description <desc>]
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Current name of the role. |
| `--new-name` | | No | New name for the role. |
| `--description` | | No | New description. |

**Example**

```bash
td role update --name data-engineer --new-name engineer
```

---

### `td role pin` / `td role unpin`

Pin a role as the default for the current session (analogous to `td collection pin`).

```bash
td role pin --name <name>
td role unpin
```

---


**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|

### `td role add-perm`

Add a permission to a role. Permissions that apply to a specific collection require the `--coll` flag; pass `<ALL>` to apply the permission to all collections.

```bash
td role add-perm --name <role> --perm <permission-type> [--coll <collection>]
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Name of the role. |
| `--perm` | | Yes* | Permission type to grant. Prompted with valid values if omitted. |
| `--coll` | | Conditional | Collection the permission applies to. Required for entity-scoped permission types. Use `<ALL>` to apply to all collections. |

**Example**

```bash
# Grant a collection-scoped permission
td role add-perm --name data-engineer --perm read --coll analytics

# Grant a permission that applies to all collections
td role add-perm --name data-engineer --perm read --coll "<ALL>"
```

---

### `td role delete-perm`

Remove a permission from a role. Identify the permission by type (and optionally collection) or by its UUID.

```bash
td role delete-perm --name <role> --perm <type> [--coll <collection>]
# or
td role delete-perm --name <role> --id <permission-id>
```

**Flags**

| Flag | Short | Required | Description |
|------|-------|----------|-------------|
| `--name` | `-n` | Yes* | Name of the role. |
| `--id` | | No | UUID of the permission to delete. Mutually exclusive with `--perm` and `--coll`. |
| `--perm` | | No | Permission type to remove. Mutually exclusive with `--id`. |
| `--coll` | | No | Collection scope. Mutually exclusive with `--id`. |

**Example**

```bash
td role delete-perm --name data-engineer --perm read --coll analytics
```

---

### `td role list-perm`

List all permissions of a role.

```bash
td role list-perm --name <role>
```

Prints a table with columns: **ID**, **Permission type**, **Entity name**, **Granted on**, **Granted by**.

**Example**

```bash
td role list-perm --name data-engineer
```

## Common workflow: create a user and assign a role

```bash
# 1. Create a user
td user create --name bob --full-name "Bob Jones" --email bob@example.com

# 2. Create a role with a collection-scoped read permission
td role create --name analyst --description "Read-only analyst role"
td role add-perm --name analyst --perm read --coll analytics

# 3. Add the user to the role
td role add-user --name analyst --user bob

# 4. Verify
td role list-user --name analyst
td role list-perm --name analyst
```
