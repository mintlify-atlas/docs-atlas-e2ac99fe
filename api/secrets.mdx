Tabsdata provides a secret system that lets you pass sensitive values—API keys, passwords, connection strings—to connectors without ever embedding them in source code or function bundles. Instead of a raw string, you supply a `Secret` object that tells the server *where* to find the value at execution time.

<Warning>
Never hardcode passwords, API keys, or access tokens as plain strings in your functions. Use `EnvironmentSecret` or `HashiCorpSecret` instead. The raw credential value is only resolved on the Tabsdata server immediately before the function runs, and is never logged or persisted.
</Warning>

---

## `EnvironmentSecret`

Reads a secret from a server-side environment variable at function execution time. This is the simplest and most common secret type—set the variable in your deployment environment and reference it by name in your code.

```python
import tabsdata as td

password = td.EnvironmentSecret("DB_PASSWORD")
```

<ParamField path="environment_variable_name" type="str" required>
  The name of the environment variable to read on the Tabsdata server when the
  function executes. The variable must be set in the server's environment; it
  does **not** need to be set in your local development environment when
  registering the function.
</ParamField>

**Full example**:

```python
import tabsdata as td

@td.publisher(
    source=td.MySQLSource(
        uri="mysql://prod-db:3306/analytics",
        query="SELECT * FROM events WHERE date > :since",
        credentials=td.UserPasswordCredentials(
            user=td.EnvironmentSecret("MYSQL_USER"),
            password=td.EnvironmentSecret("MYSQL_PASSWORD"),
        ),
        initial_values={"since": "2024-01-01"},
    ),
    tables="analytics/events",
)
def load_events(tf: td.TableFrame) -> td.TableFrame:
    return tf
```

## `HashiCorpSecret`

Reads a secret from a HashiCorp Vault KV store. The Tabsdata server must be configured with the Vault URL and token via environment variables.

```python
import tabsdata as td

api_key = td.HashiCorpSecret(
    path="secret/my-app",
    name="api_key",
)
```

<ParamField path="path" type="str" required>
  The path to the secret in HashiCorp Vault KV v2, e.g. `"secret/my-app"`.
</ParamField>

<ParamField path="name" type="str" required>
  The key name within the secret at `path`. For a Vault secret with fields
  `{"api_key": "abc123", "region": "us-east"}`, pass `name="api_key"` to
  retrieve `"abc123"`.
</ParamField>

<ParamField path="vault" type="str" default='"HASHICORP"'>
  The logical vault name, used to resolve environment variables on the server.
  The vault name must contain only uppercase letters, numbers, and underscores,
  and must not begin with a digit.

  For a vault named `HASHICORP` (the default), the server reads:
  - `TDS_HASHICORP_URL` — the Vault server URL
  - `TDS_HASHICORP_TOKEN` — the Vault access token
  - `TDS_HASHICORP_NAMESPACE` — optional Vault namespace

  For a custom vault named `PROD_VAULT`, the server reads:
  - `TDS_PROD_VAULT_URL`
  - `TDS_PROD_VAULT_TOKEN`
  - `TDS_PROD_VAULT_NAMESPACE`
</ParamField>

**Full example with a custom vault**:

```python
import tabsdata as td

db_pass = td.HashiCorpSecret(
    path="secret/databases/postgres",
    name="password",
    vault="PROD_VAULT",
)

@td.subscriber(
    tables="analytics/summary",
    destination=td.PostgresDestination(
        uri="postgresql://prod-pg:5432/reporting",
        destination_table="daily_summary",
        credentials=td.UserPasswordCredentials(
            user="reporting_writer",
            password=db_pass,
        ),
    ),
)
def export_summary(tf: td.TableFrame) -> td.TableFrame:
    return tf
```

## `Secret` base class

`Secret` is an abstract base class. Subclass it only if you are building a custom secret backend.

```python
from tabsdata._secret import Secret

class MyCustomSecret(Secret):
    def _to_dict(self) -> dict: ...

    @property
    def secret_value(self) -> str:
        # Resolve and return the raw string value.
        ...
```

The `.secret_value` property is called by the Tabsdata server at execution time. It must return the raw string credential.

## Evaluation timing

<Note>
Secrets are **not** evaluated when you call the decorator or when you register the function with `td fn register`. They are resolved only on the Tabsdata server, immediately before the function executes each run. This means:

- Your local Python environment does not need access to production secrets during development.
- Rotating a secret in your environment or Vault automatically takes effect on the next function run without re-registering.
</Note>

## Mixing plain strings and secrets

All credential parameters that accept `str | Secret` automatically wrap a plain string in a `DirectSecret` internally. This means you can still use plain strings during local testing:

```python
# OK for local testing — avoid in production
creds = td.UserPasswordCredentials(user="admin", password="test-password")

# Recommended for all environments
creds = td.UserPasswordCredentials(
    user=td.EnvironmentSecret("DB_USER"),
    password=td.EnvironmentSecret("DB_PASSWORD"),
)
```
