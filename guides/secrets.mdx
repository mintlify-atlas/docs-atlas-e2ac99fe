Connecting to databases, object stores, and cloud services requires sensitive values such as passwords, API keys, and access tokens. Hardcoding these values in source code is a security risk: they end up in version control, logs, and error messages. Tabsdata provides a `Secret` abstraction that lets you declare *where* a value lives â€” an environment variable or HashiCorp Vault â€” without ever embedding the value itself in your function code. Secrets are only evaluated at execution time on the server, not when you register the function.

## Secret types

### `EnvironmentSecret`

Reads a value from a named environment variable at execution time on the Tabsdata server. This is the simplest option: set the variable in the server's environment (or in a `.env` file managed by your deployment tooling) and reference it by name in your code.

```python
td.EnvironmentSecret("VARIABLE_NAME")
```

<ParamField path="environment_variable_name" type="str" required>
  The name of the environment variable to read at execution time.
</ParamField>

### `HashiCorpSecret`

Reads a value from HashiCorp Vault using the KV secrets engine. At execution time the server looks up the Vault URL and token from its own environment (using the `TDS_<VAULT>_URL` and `TDS_<VAULT>_TOKEN` variables) and fetches the secret at the given path.

```python
td.HashiCorpSecret(path="secret/data/myapp", name="db_password", vault="HASHICORP")
```

<ParamField path="path" type="str" required>
  The KV path of the secret in Vault, for example `"secret/data/myapp"`.
</ParamField>

<ParamField path="name" type="str" required>
  The key within the secret document to retrieve.
</ParamField>

<ParamField path="vault" type="str" default="HASHICORP">
  The logical vault name used to locate the corresponding server-side environment variables (`TDS_<VAULT>_URL`, `TDS_<VAULT>_TOKEN`, and optionally `TDS_<VAULT>_NAMESPACE`). Must contain only uppercase letters, digits, and underscores, and must not start with a digit.
</ParamField>

<Warning>
  Never log, print, or expose the resolved value of a secret inside your function. The `secret_value` property is intended exclusively for the Tabsdata server runtime. Printing it to stdout or storing it in a Tabsdata table defeats the purpose of using a secret.
</Warning>

## Using secrets in credential objects

Secrets are passed into **credential classes**, which are then passed into source and destination classes. Every credential field that accepts a `str` also accepts any `Secret` subclass.

### `UserPasswordCredentials` â€” SQL databases

Use this for MySQL, PostgreSQL, Oracle, and MariaDB connections.

```python
import tabsdata as td

credentials = td.UserPasswordCredentials(
    user="etl_user",
    password=td.EnvironmentSecret("DB_PASSWORD"),
)

@td.publisher(
    source=td.MySQLSource(
        uri="mysql://db.example.com:3306/warehouse",
        query="SELECT * FROM events WHERE processed = 0",
        credentials=credentials,
    ),
    tables="raw_events",
)
def ingest_events(events: td.TableFrame) -> td.TableFrame:
    return events
```

### `S3AccessKeyCredentials` â€” Amazon S3

Use this for AWS S3 sources and destinations. `S3AccessKeyCredentials` is the concrete credential class for `S3Source` and `S3Destination`.

```python
import tabsdata as td


@td.publisher(
    source=td.S3Source(
        uri="s3://my-bucket/raw/*.parquet",
        credentials=td.S3AccessKeyCredentials(
            aws_access_key_id=td.EnvironmentSecret("AWS_ACCESS_KEY_ID"),
            aws_secret_access_key=td.EnvironmentSecret("AWS_SECRET_ACCESS_KEY"),
        ),
        region="us-east-1",
    ),
    tables="raw_data",
    trigger_by=td.CronTrigger("0 6 * * *"),
)
def ingest_s3(raw: td.TableFrame) -> td.TableFrame:
    return raw
```

### `HashiCorpSecret` with S3 credentials

When your organisation uses HashiCorp Vault, swap `EnvironmentSecret` for `HashiCorpSecret` â€” the rest of the code is identical.

```python
import tabsdata as td


@td.subscriber(
    tables="curated",
    destination=td.S3Destination(
        uri="s3://my-bucket/curated/output.parquet",
        credentials=td.S3AccessKeyCredentials(
            aws_access_key_id=td.HashiCorpSecret(
                path="secret/data/aws",
                name="access_key_id",
            ),
            aws_secret_access_key=td.HashiCorpSecret(
                path="secret/data/aws",
                name="secret_access_key",
            ),
        ),
        region="eu-west-1",
    ),
)
def export_curated(curated: td.TableFrame) -> td.TableFrame:
    return curated
```

### `AzureAccountKeyCredentials` â€” Azure Blob Storage

```python
import tabsdata as td

azure_creds = td.AzureAccountKeyCredentials(
    account_name=td.EnvironmentSecret("AZURE_ACCOUNT_NAME"),
    account_key=td.EnvironmentSecret("AZURE_ACCOUNT_KEY"),
)

@td.publisher(
    source=td.AzureSource(
        uri="az://my-container/data/snapshot.csv",
        credentials=azure_creds,
    ),
    tables="azure_snapshot",
)
def ingest_azure(snapshot: td.TableFrame) -> td.TableFrame:
    return snapshot
```

### `GCPServiceAccountKeyCredentials` â€” Google Cloud Storage

```python
import tabsdata as td

gcp_creds = td.GCPServiceAccountKeyCredentials(
    service_account_key=td.EnvironmentSecret("GCP_SERVICE_ACCOUNT_JSON"),
)

@td.publisher(
    source=td.GCSSource(
        uri="gs://my-bucket/data/*.parquet",
        credentials=gcp_creds,
    ),
    tables="gcs_data",
)
def ingest_gcs(data: td.TableFrame) -> td.TableFrame:
    return data
```

## Credential class reference

| Credential class | Accepted by | Fields |
|------------------|-------------|--------|
| `UserPasswordCredentials(user, password)` | `MySQLSource/Destination`, `PostgresSource/Destination`, `OracleSource/Destination`, `MariaDBSource/Destination` | `user`, `password` |
| `S3AccessKeyCredentials(aws_access_key_id, aws_secret_access_key)` | `S3Source`, `S3Destination` | `aws_access_key_id`, `aws_secret_access_key` |
| `HMACCredentials(access_key, secret_key)` | Cloud warehouse staging (ClickHouse, Druid, StarRocks) | `access_key`, `secret_key` |
| `AzureAccountKeyCredentials(account_name, account_key)` | `AzureSource`, `AzureDestination` | `account_name`, `account_key` |
| `GCPServiceAccountKeyCredentials(service_account_key)` | `GCSSource`, `GCSDestination` | `service_account_key` |

<Note>
  Secrets are resolved **only at function execution time** on the Tabsdata server. When you register a function with `td fn register`, the secret object is serialised as a reference (for example, `{"environment-secret": {"environment_variable_name": "DB_PASSWORD"}}`) â€” the actual value is never read or transmitted during registration.
</Note>
