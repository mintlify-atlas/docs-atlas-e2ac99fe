Tabsdata's plugin API lets you write custom connectors in pure Python. Any data source or destination you can read from or write to programmatically can be integrated using the `SourcePlugin` and `DestinationPlugin` base classes. Custom plugins are passed directly as the `source` or `destination` parameter to the `@publisher` and `@subscriber` decorators, just like the built-in connectors.

<Tip>
Use custom plugins to integrate proprietary APIs, on-premise systems, obscure file formats, or any external data store that isn't covered by Tabsdata's built-in connectors. The framework handles scheduling, retries, and state management — you only implement the I/O logic.
</Tip>

## `SourcePlugin` — custom publisher source

Subclass `SourcePlugin` and implement `chunk(working_dir)` to pull data from any external system and expose it as one or more TableFrames to your publisher function.

### How it works

<Steps>
  <Step title="Subclass SourcePlugin">
    Import `SourcePlugin` from `tabsdata` and create a subclass.
  </Step>
  <Step title="Implement chunk()">
    Download or query your data, save each logical dataset as a Parquet file inside `working_dir`, and return a list of file paths. Each path in the returned list maps to one positional argument of the publisher function, in the same order.
  </Step>
  <Step title="Pass to @publisher">
    Pass an instance of your plugin as the `source` parameter of `@td.publisher`.
  </Step>
</Steps>

### Interface

```python
class SourcePlugin:
    def chunk(
        self,
        working_dir: str,
    ) -> Union[str, Tuple[str, ...], List[str]]:
        """
        Download data and save it as Parquet files.

        Args:
            working_dir: Folder where Parquet files must be saved.

        Returns:
            A list of file paths (relative to working_dir or absolute).
            Each element maps to one positional argument of the publisher
            function. Return a nested list to provide multiple files for
            a single argument, e.g.:
              ["file1.parquet", ["file2.parquet", "file3.parquet"]]
        """
        raise NotImplementedError

    @property
    def initial_values(self) -> dict:
        """
        State persisted between runs. Override to support incremental loading.
        """
        return {}
```

### Complete SourcePlugin example

The example below fetches paginated data from an HTTP REST API, saves each page as a Parquet file, and maps all pages as a single argument to the publisher function.

```python
import os
import polars as pl
import requests
import tabsdata as td
from tabsdata import SourcePlugin


class RestApiSource(SourcePlugin):
    """Fetches records from a paginated REST API."""

    def __init__(self, base_url: str, api_key: str, page_size: int = 500):
        self.base_url = base_url
        self.api_key = api_key
        self.page_size = page_size

    def chunk(self, working_dir: str):
        headers = {"Authorization": f"Bearer {self.api_key}"}
        page = 1
        file_paths = []

        while True:
            resp = requests.get(
                f"{self.base_url}/records",
                headers=headers,
                params={"page": page, "size": self.page_size},
                timeout=30,
            )
            resp.raise_for_status()
            records = resp.json().get("data", [])
            if not records:
                break

            df = pl.DataFrame(records)
            file_path = os.path.join(working_dir, f"page_{page}.parquet")
            df.write_parquet(file_path)
            file_paths.append(file_path)
            page += 1

        # Return a nested list so the publisher receives all pages as
        # one argument (a list of TableFrames).
        return [file_paths]


# Use the plugin in a publisher
@td.publisher(
    source=RestApiSource(
        base_url="https://api.example.com/v1",
        api_key="my-secret-api-key",
    ),
    tables="myorg/api/records",
)
def ingest_api_records(pages: list[td.TableFrame]) -> td.TableFrame:
    # Concatenate all pages into a single TableFrame
    import polars as pl
    combined = pl.concat([p.collect() for p in pages]).lazy()
    return td.TableFrame(combined)
```

### Incremental loading with `initial_values`

Persist state between runs by overriding the `initial_values` property. The framework reads `initial_values` before each run and writes any updated value back to persistent storage after the run.

```python
class IncrementalApiSource(SourcePlugin):
    def __init__(self, base_url: str, api_key: str):
        self.base_url = base_url
        self.api_key = api_key
        # Initial watermark — overwritten by the stored offset on subsequent runs
        self._initial_values = {"since": "1970-01-01T00:00:00Z"}

    @property
    def initial_values(self) -> dict:
        return self._initial_values

    @initial_values.setter
    def initial_values(self, values: dict):
        self._initial_values = values

    def chunk(self, working_dir: str):
        since = self.initial_values.get("since", "1970-01-01T00:00:00Z")
        resp = requests.get(
            f"{self.base_url}/events",
            params={"since": since},
            timeout=30,
        )
        resp.raise_for_status()
        records = resp.json()

        if not records:
            return [None]  # Signal that there is no new data

        df = pl.DataFrame(records)
        file_path = os.path.join(working_dir, "events.parquet")
        df.write_parquet(file_path)

        # Advance the watermark to the maximum timestamp seen
        latest = df["updated_at"].max()
        self.initial_values = {"since": latest}

        return [file_path]
```

## `DestinationPlugin` — custom subscriber destination

Subclass `DestinationPlugin` and implement `stream(working_dir, *results)` to write the TableFrame data produced by your subscriber function to any external system.

### Interface

```python
class DestinationPlugin:
    def stream(
        self,
        working_dir: str,
        *results,           # polars LazyFrame | None, one per output TableFrame
    ) -> None:
        """
        Write the subscriber results to the target system.

        Args:
            working_dir: Folder for temporary intermediate files.
            results: One positional argument per output TableFrame from the
                     subscriber function. Each may be a polars LazyFrame or None.
        """
        # Default: calls chunk() then write().
        files = self.chunk(working_dir, *results)
        self.write(files)

    def chunk(
        self,
        working_dir: str,
        *results,
    ) -> list:
        """
        Materialise results to local Parquet files (intermediate step).
        Override if you need a two-phase commit pattern.
        """
        raise NotImplementedError

    def write(self, files: list) -> None:
        """
        Upload the local intermediate files to the final destination.
        Override together with chunk() for the two-phase pattern.
        """
```

For most use cases, overriding only `stream()` is sufficient.

### Complete DestinationPlugin example

```python
import os
import polars as pl
import requests
import tabsdata as td
from tabsdata import DestinationPlugin


class WebhookDestination(DestinationPlugin):
    """Posts each row of a TableFrame as a JSON payload to a webhook URL."""

    def __init__(self, webhook_url: str, batch_size: int = 100):
        self.webhook_url = webhook_url
        self.batch_size = batch_size

    def stream(self, working_dir: str, *results) -> None:
        for result in results:
            if result is None:
                continue

            # Materialise the LazyFrame
            df: pl.DataFrame = result.collect()
            records = df.to_dicts()

            # Send in batches
            for i in range(0, len(records), self.batch_size):
                batch = records[i : i + self.batch_size]
                resp = requests.post(
                    self.webhook_url,
                    json={"records": batch},
                    timeout=30,
                )
                resp.raise_for_status()


# Use the plugin in a subscriber
@td.subscriber(
    tables="myorg/alerts/triggered",
    destination=WebhookDestination(
        webhook_url="https://hooks.example.com/alerts",
        batch_size=50,
    ),
)
def forward_alerts(alerts: td.TableFrame) -> td.TableFrame:
    return alerts.filter(td.col("severity") == "critical")
```

### Two-phase chunk / write pattern

For destinations that benefit from separating the materialisation step (writing local Parquet files) from the upload step (pushing to the remote system), override `chunk()` and `write()` instead of `stream()`:

```python
class S3CompatibleDestination(DestinationPlugin):
    def __init__(self, bucket: str, prefix: str):
        self.bucket = bucket
        self.prefix = prefix

    def chunk(self, working_dir: str, *results) -> list[str | None]:
        """Materialise each result to a local Parquet file."""
        paths = []
        for idx, result in enumerate(results):
            if result is None:
                paths.append(None)
            else:
                path = os.path.join(working_dir, f"output_{idx}.parquet")
                result.sink_parquet(path)
                paths.append(path)
        return paths

    def write(self, files: list[str | None]) -> None:
        """Upload each Parquet file to an S3-compatible store."""
        import boto3
        s3 = boto3.client("s3")
        for file_path in files:
            if file_path is None:
                continue
            key = f"{self.prefix}/{os.path.basename(file_path)}"
            s3.upload_file(file_path, self.bucket, key)
```

## Registering plugins with decorators

Custom plugins are used identically to built-in connectors:

```python
# Source plugin → @publisher
@td.publisher(
    source=MyCustomSource(...),
    tables="myorg/collection/table",
)
def my_publisher(data: td.TableFrame) -> td.TableFrame:
    ...

# Destination plugin → @subscriber
@td.subscriber(
    tables="myorg/collection/table",
    destination=MyCustomDestination(...),
)
def my_subscriber(data: td.TableFrame) -> td.TableFrame:
    ...
```

<Note>
The `source` and `destination` parameters also accept the built-in connector classes (e.g., `S3Source`, `SnowflakeDestination`). Custom plugins and built-in connectors are interchangeable wherever these parameters appear. See [API: Inputs](/api/inputs) and [API: Outputs](/api/outputs) for the full decorator API reference.
</Note>
