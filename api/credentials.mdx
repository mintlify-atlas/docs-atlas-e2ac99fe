Credential objects encapsulate the authentication information needed by file-source and SQL-destination plugins. Every credential parameter can accept either a plain `str` or a `Secret` instance. In production, always use a `Secret` (such as `EnvironmentSecret`) rather than hardcoding sensitive values in source code.

<Tip>
Wrap any credential field with `td.EnvironmentSecret("VAR_NAME")` or `td.HashiCorpSecret(...)` to keep secrets out of your code and function bundles. See [Secrets](/api/secrets).
</Tip>

---

## `UserPasswordCredentials`

Username and password authentication. Used by all SQL connectors (MySQL, PostgreSQL, Oracle, MariaDB) and any compatible service.

```python
import tabsdata as td

creds = td.UserPasswordCredentials(
    user=td.EnvironmentSecret("DB_USER"),
    password=td.EnvironmentSecret("DB_PASSWORD"),
)
```

<ParamField path="user" type="str | Secret" required>
  The database username. Accepts a plain string or any `Secret` subclass.
</ParamField>

<ParamField path="password" type="str | Secret" required>
  The database password. Accepts a plain string or any `Secret` subclass.
</ParamField>

## `S3AccessKeyCredentials`

AWS IAM access-key credentials for S3 sources and destinations.

```python
creds = td.S3AccessKeyCredentials(
    aws_access_key_id=td.EnvironmentSecret("AWS_ACCESS_KEY_ID"),
    aws_secret_access_key=td.EnvironmentSecret("AWS_SECRET_ACCESS_KEY"),
)
```

<ParamField path="aws_access_key_id" type="str | Secret" required>
  The AWS access key ID (begins with `AKIA…`).
</ParamField>

<ParamField path="aws_secret_access_key" type="str | Secret" required>
  The AWS secret access key corresponding to the access key ID.
</ParamField>

---

## `AzureAccountKeyCredentials`

Account name and account key authentication for Azure Blob Storage.

```python
creds = td.AzureAccountKeyCredentials(
    account_name=td.EnvironmentSecret("AZURE_STORAGE_ACCOUNT"),
    account_key=td.EnvironmentSecret("AZURE_STORAGE_KEY"),
)
```

<ParamField path="account_name" type="str | Secret" required>
  The Azure Storage account name.
</ParamField>

<ParamField path="account_key" type="str | Secret" required>
  The Azure Storage account key (base-64 encoded).
</ParamField>

---

## `GCPServiceAccountKeyCredentials`

Service account key authentication for Google Cloud Storage.

```python
creds = td.GCPServiceAccountKeyCredentials(
    service_account_key=td.EnvironmentSecret("GCP_SERVICE_ACCOUNT_JSON"),
)
```

<ParamField path="service_account_key" type="str | Secret" required>
  The full JSON content of a GCP service account key file, or a `Secret` that
  resolves to that JSON string at execution time.
</ParamField>

---

## `HMACCredentials`

Generic HMAC (Hash-based Message Authentication Code) credentials. Use this for any S3-compatible service that uses HMAC-style access keys, such as GCS HMAC keys, MinIO, or Ceph.

```python
creds = td.HMACCredentials(
    access_key=td.EnvironmentSecret("HMAC_ACCESS_KEY"),
    secret_key=td.EnvironmentSecret("HMAC_SECRET_KEY"),
)
```

<ParamField path="access_key" type="str | Secret" required>
  The HMAC access key ID.
</ParamField>

<ParamField path="secret_key" type="str | Secret" required>
  The HMAC secret key.
</ParamField>

## Using secrets with credentials

Every credential parameter accepts a `Secret` instance. The secret value is resolved at function execution time on the server, not when the function is registered, so your code never needs to handle the raw credential value.

```python
import tabsdata as td

@td.publisher(
    source=td.S3Source(
        uri="s3://my-bucket/data/*.parquet",
        credentials=td.S3AccessKeyCredentials(
            aws_access_key_id=td.EnvironmentSecret("AWS_KEY_ID"),
            aws_secret_access_key=td.EnvironmentSecret("AWS_SECRET"),
        ),
        region="us-east-1",
    ),
    tables="my_collection/raw_data",
)
def ingest_s3(tf: td.TableFrame) -> td.TableFrame:
    return tf
```

See [Secrets](/api/secrets) for all available secret types, including `HashiCorpSecret` for reading from HashiCorp Vault.

## Abstract base classes

The concrete credential classes extend these abstract bases, which you can use for type-checking:

| Base class | Used by |
|------------|---------|
| `S3Credentials` | `S3AccessKeyCredentials`, `HMACCredentials` |
| `AzureCredentials` | `AzureAccountKeyCredentials` |
| `GCPCredentials` | `GCPServiceAccountKeyCredentials` |
